# EDUQG 内容建设方案

## 一、需求背景/目标

EDUQG 是面向教育场景的智能题目生成系统，核心目标是帮助教师、教研人员和教学平台快速生成高质量、可评估、可迭代的试题内容。

在日常教学、作业布置、阶段测评和题库建设中，教师经常需要围绕指定知识点、难度层级、题型和学生水平设计题目。传统人工出题存在耗时长、难度控制不稳定、题目质量依赖个人经验、配图题制作成本高、生成后缺少质量检查等问题。尤其在数学、物理等 STEM 学科中，很多题目需要依赖几何图形、函数图像、受力图、电路图或过程示意图，仅靠文本生成很难保证题目条件、图示信息和解题过程的一致性。

EDUQG 的建设目标是形成一套从教师需求到最终题目输出的完整智能出题流程。系统能够根据教师输入的学科、知识点、难度、题型、内容模式和图片要求，自动生成符合教学目标的题目，并通过评价、修订和渲染机制提升题目可用性。

项目目标包括：

1. 提升教师出题效率，减少重复性命题和题库维护工作。
2. 保证题目与知识点、难度、题型和教学目标严格对齐。
3. 支持文本题和图片题，图片必须服务于作答或解析，而不是装饰性配图。
4. 引入生成后评价机制，检查答案正确性、解析完整性、难度匹配度和图文一致性。
5. 支持多种出题策略，如 Direct、CoT、ReAct、DEAR、EQPR、EvoQ，以适应不同质量和效率要求。
6. 沉淀教师画像、学生画像和生成历史，为个性化出题和后续教学分析提供依据。

## 二、应用场景/用户

EDUQG 的主要用户包括一线教师、教研人员、教育平台运营人员、自适应学习系统和学生自学系统。

### 1. 一线教师

教师可以使用 EDUQG 快速生成课堂练习、课后作业、单元测验、随堂检测、错题变式和分层练习题。教师只需要明确知识点、难度和题型，系统即可生成题干、选项、答案和解析。

### 2. 教研人员

教研人员可以使用 EDUQG 批量建设题库，围绕课程标准和教学进度生成不同知识点、不同难度梯度的题目，提升题库覆盖度和建设效率。

### 3. 教育平台

在线教育平台可以将 EDUQG 接入题库系统、练习系统、智能作业系统或 AI 伴学系统中，用于自动补充题目、生成个性化练习、构建阶段性测评内容。

### 4. 学生自学系统

系统可以根据学生的能力水平、知识掌握情况和历史错题，为学生生成针对性练习，帮助学生查漏补缺。

### 5. 测评与诊断系统

EDUQG 可以根据指定知识点和能力要求生成诊断题，用于判断学生是否掌握核心概念、是否存在常见误区，以及是否需要进一步讲解或练习。

典型应用场景包括：

1. 指定知识点生成题目：如“初中数学、勾股定理、难度 3、选择题”。
2. 生成图文结合题：如几何图形题、函数图像题、受力分析题、电路图题。
3. 错题变式生成：基于学生错题生成同类但不重复的新题。
4. 分层练习生成：针对基础、中等和优秀学生生成不同难度题目。
5. 自然语言出题：教师用自然语言描述需求，系统自动整理成结构化出题规范。
6. 题库批量建设：围绕某一章节或知识点集合批量生成题目。

## 三、输入输出

EDUQG 的输入可以分为教师显式输入、系统画像输入和生成控制参数。

### 输入内容

1. 学科
   如数学、物理、化学、生物等。

2. 知识点
   如勾股定理、一次函数、牛顿第二定律、受力分析、电路串并联等。

3. 难度
   支持 1-6 等级，也可兼容 easy、medium、hard 等难度描述。

4. 题型
   当前支持选择题、判断题、简答题。

5. 内容模式
   支持纯文本题和图片题。

6. 图片要求
   包括是否配图、是否必须配图、图片出现在题干/选项/解析中，以及图片是否必须参与作答。

7. 出题算法
   支持 direct、cot、react、dear、eqpr、evoq 等策略。

8. 教师画像
   包括教师偏好的教学风格、语言风格、难度控制方式、可视化偏好和命题约束。

9. 学生画像
   包括学生能力水平、知识掌握情况、常见错误、学习偏好和历史表现。

10. 附加要求
   如“题目不要太长”“适合课堂小测”“增加迷惑性选项”“解析要分步骤”等。

### 输出内容

1. 题干
   生成符合学科、知识点和教学目标的题目文本。

2. 选项
   对选择题生成 A/B/C/D 四个选项，并保证选项格式合法、答案唯一。

3. 标准答案
   输出选择题答案、判断题答案或简答题关键答案。

4. 解题步骤
   输出完整解析，说明解题思路、关键公式、推导过程和最终结论。

5. 图片资源
   对图片题输出题干配图、选项配图或解析配图，图片内容需要与题目条件一致。

6. 结构化结果
   输出统一的 JSON 结构，便于前端展示、题库存储、质量分析和后续调用。

7. 生成状态
   返回 request、generate、evaluate、render 等阶段状态，方便前端展示进度。

8. 评价结果
   输出题目质量、难度匹配、答案正确性、图文一致性、教学有效性等评价信息。

9. 历史记录
   保存生成成功结果、失败原因、评价意见和用户调整记录，便于后续复用。

## 四、评价结果

EDUQG 不只是生成题目，还需要对生成结果进行质量评价。评价机制用于判断题目是否真正可用，避免出现答案错误、难度偏差、题干歧义、选项重复、解析不完整、图片无关等问题。

评价维度包括：

1. 知识点匹配度
   判断题目是否围绕指定知识点展开，是否偏题、超纲或考查点不明确。

2. 难度匹配度
   判断题目是否符合教师设定的难度层级，是否过难或过易。

3. 题型合规性
   选择题必须有四个选项和唯一正确答案；判断题答案必须明确；简答题需要有可判定的标准答案。

4. 答案正确性
   检查标准答案是否正确，解析过程是否能够推出该答案。

5. 解析完整性
   判断解题步骤是否清晰、完整、适合学生理解。

6. 教学有效性
   判断题目是否能够有效考查目标知识点，是否具有练习、诊断或测评价值。

7. 语言可读性
   检查题干、选项和解析是否表达清楚，是否存在歧义或表述不自然。

8. 图文一致性
   对图片题检查图示是否与题干条件一致，是否包含解题所需信息。

9. 图片有效性
   判断图片是否真正参与作答，避免生成与解题无关的装饰性图片。

10. 渲染可用性
   对图片题检查 SVG 或 Manim 图示是否能够正常渲染，是否存在裁切、遮挡、布局混乱等问题。

评价结果分为三类：

1. 通过
   题目满足出题要求，可直接展示、入库或发给学生练习。

2. 需修改
   题目基本可用，但存在局部问题，需要根据评价意见自动修订或人工调整。

3. 不通过
   题目存在关键错误，如答案错误、知识点不匹配、图示错误、难度严重偏差等，需要重新生成。

评价结果可进一步量化为评分：

- 90 分以上：高质量题目，可直接使用。
- 75-89 分：整体可用，建议局部优化。
- 60-74 分：基本成型，但需要人工复核。
- 60 分以下：质量不足，建议重新生成。

## 五、整体工作流程/设计方案

EDUQG 的整体流程可以概括为：

教师需求输入 → 意图识别 → 出题规范确认 → 智能体路由 → 题目生成 → 自动评价 → 修订优化 → 图片渲染 → 最终输出 → 画像沉淀

### 1. 教师需求输入

教师可以通过自然语言或表单输入出题要求。例如：

“请生成一道初中数学题，知识点是勾股定理，难度 3，题型为选择题，需要题干配图。”

### 2. 意图识别与字段抽取

系统识别教师意图，抽取学科、知识点、难度、题型、内容模式、图片要求等关键字段。如果信息不完整，系统需要继续追问，例如“请选择题型”或“是否需要配图”。

### 3. 出题规范生成

系统将教师需求转化为标准化的 `edu-question-spec.v1`。该规范明确题目生成所需的关键字段，并区分教师控制字段和系统生成字段。

教师控制字段包括：

1. 学科
2. 知识点
3. 难度
4. 题型
5. 内容模式
6. 出题算法
7. 图片要求

系统生成字段包括：

1. 生成计划
2. 题目草稿
3. 解题步骤
4. 评价意见
5. 修订建议
6. 画像更新建议

### 4. 教师确认

系统在正式生成前要求教师确认关键字段，避免 AI 擅自修改学科、知识点、难度、题型或图片要求。只有当出题规范状态为 ready 时，系统才进入生成流程。

### 5. 智能体路由

系统根据内容模式和算法选择对应智能体：

1. 文本题进入文本题生成智能体。
2. 图片题进入图文题生成智能体。
3. 文本题由文本题评价智能体检查质量。
4. 图片题由图文题评价智能体检查图文一致性和图片有效性。
5. EvoQ 等复杂算法会调用专门的进化式出题策略。

### 6. 题目草稿生成

生成器根据出题规范生成题干、选项、答案和解析。对于图片题，还会生成图片位置、图片内容和可渲染图示资源。

### 7. 自动评价

评价智能体对题目进行检查，包括结构合法性、答案正确性、难度匹配、知识点匹配、解析完整性、图文一致性和图片可用性。

### 8. 反馈修订

如果评价不通过，系统根据评价意见进行自动修订。修订后再次评价，直到通过或达到重试上限。

### 9. 图片生成与渲染

对图片题，系统会生成安全可控的 SVG 或 Manim 图示，并完成真实图片渲染。必需图片题只有在图片成功生成并与题目相关时才算完成。

### 10. 最终结果组装

系统将题干、选项、答案、解析、图片资源、评价信息和生成元数据组装成统一响应，供前端展示或题库存储。

### 11. 结果保存与画像更新

系统保存生成结果、失败原因、评价信息和用户反馈，并用于后续教师偏好学习、学生能力建模和个性化推荐。

## 六、案例

### 案例一：初中数学图文选择题

教师输入：

> 请生成一道初中数学题，知识点是勾股定理，难度 3，题型为选择题，需要题干配图。

系统识别：

| 字段 | 结果 |
|---|---|
| 学科 | 数学 |
| 知识点 | 勾股定理 |
| 难度 | 3 |
| 题型 | 选择题 |
| 内容模式 | 图片题 |
| 图片位置 | 题干配图 |
| 图片要求 | 图片必须参与作答 |
| 生成算法 | direct 或 evoq |

生成题目示例：

题干：

如图，在直角三角形 ABC 中，∠C = 90°，AC = 6，BC = 8，求 AB 的长度。

选项：

A. 8
B. 10
C. 12
D. 14

答案：

B

解析：

因为三角形 ABC 是直角三角形，且 AC 和 BC 是两条直角边。根据勾股定理：

AB² = AC² + BC²
AB² = 6² + 8² = 36 + 64 = 100
所以 AB = 10。

因此正确答案是 B。

系统评价：

1. 知识点匹配：通过，题目直接考查勾股定理。
2. 难度匹配：通过，适合基础到中等难度练习。
3. 题型合规：通过，四个选项且答案唯一。
4. 图片相关性：通过，图示提供直角三角形和边长信息。
5. 解析完整性：通过，解题步骤清晰。
6. 最终状态：可展示、可入库。

### 案例二：物理受力分析题

教师输入：

> 生成一道初中物理受力分析题，难度中等，需要图示，题目用于课堂练习。

系统生成方向：

1. 构造一个物体在水平面上受到拉力和摩擦力的情境。
2. 在图中标注重力、支持力、拉力和摩擦力方向。
3. 让学生判断物体所受力的方向或合力情况。
4. 解析中说明各力的来源和方向。

评价重点：

1. 图中力的方向是否正确。
2. 题干是否明确说明运动状态或受力条件。
3. 答案是否唯一。
4. 解析是否符合初中物理知识范围。

### 案例三：错题变式生成

教师输入：

> 学生在一次函数图像题上经常混淆斜率和截距，请生成一道变式题，难度 4，选择题。

系统生成方向：

1. 围绕一次函数表达式和图像特征设计题目。
2. 设置干扰项，区分斜率、截距、函数增减性。
3. 在解析中指出常见错误原因。
4. 根据学生画像标记该题适合用于薄弱点巩固。

## 七、使用手册/使用说明

### 1. 创建出题任务

教师进入 EDUQG 出题页面后，可以选择自然语言输入，也可以通过表单填写需求。

### 2. 填写核心信息

建议至少填写以下信息：

1. 学科
2. 知识点
3. 难度
4. 题型
5. 内容模式

示例：

“数学、一次函数、难度 4、选择题、文本题。”

### 3. 选择是否配图

如果是普通文本题，选择“无图”。
如果题目需要图形、函数图像、表格、流程图或示意图，选择“可选配图”或“必须配图”。

图片模式包括：

1. 无图：只生成纯文本题。
2. 可选配图：系统判断是否需要配图。
3. 必须配图：系统必须生成与作答相关的图片。

### 4. 确认图片位置

图片可以放在：

1. 题干中：适合几何题、函数图像题、受力分析题。
2. 选项中：适合图形选择题、图像辨析题。
3. 解析中：适合需要过程图或辅助说明的题目。

### 5. 选择生成算法

系统支持多种生成策略：

1. Direct：快速生成，适合普通题目。
2. CoT：强调分步推理，适合需要明确解题思路的题目。
3. ReAct：适合需要推理、检查和调整的题目。
4. DEAR：适合需要分解、分析和修订的复杂题目。
5. EQPR：适合强调评价、处理和优化的题目生成。
6. EvoQ：适合通过多候选、多轮优化获得更高质量题目。

### 6. 确认出题规范

系统会整理出结构化出题规范。教师需要确认关键字段，确认后才进入生成流程。若缺少必要字段，系统会提示补充。

### 7. 查看生成进度

系统会展示生成阶段状态：

1. request：请求已接收。
2. generate：正在生成题目草稿。
3. evaluate：正在评价题目质量。
4. render：正在组装结果或渲染图片。

### 8. 查看生成结果

生成完成后，教师可以查看：

1. 题干
2. 选项
3. 答案
4. 解析
5. 图片
6. 评价结果
7. 生成状态

### 9. 修改和重新生成

如果题目不符合预期，教师可以调整：

1. 知识点
2. 难度
3. 题型
4. 图片要求
5. 生成算法
6. 语言风格
7. 使用场景

调整后可重新生成。

### 10. 保存与入库

教师确认后，题目可以保存到题库，也可以用于课堂练习、作业、测验或个性化推荐。

### 11. 使用建议

为了获得更好的生成效果，建议教师输入时尽量明确：

1. 目标学生是谁。
2. 题目用于课堂练习、作业还是测验。
3. 需要考查哪个具体知识点。
4. 难度希望偏基础、中等还是提升。
5. 是否必须使用图示。
6. 是否需要设置常见误区或干扰项。

## 总结

EDUQG 的核心价值在于将教师的出题需求转化为标准化、可评价、可渲染、可沉淀的题目生成流程。它不是简单的文本生成工具，而是一个围绕教育题目生产全过程设计的智能系统，覆盖需求理解、规范确认、题目生成、自动评价、图文渲染、结果输出和画像沉淀。

通过 EDUQG，教师可以更高效地完成出题任务，教育平台可以更低成本地扩展题库，学生也可以获得更贴合自身学习情况的个性化练习内容。
